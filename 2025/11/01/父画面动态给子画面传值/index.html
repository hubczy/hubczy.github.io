父画面动态给子画面传值

Posted on 2025-11-01

父画面

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
    <head>
        <title>select标签multiple属性的使用方法</title>
        <meta http-equiv="Content-Type" content="text/html; charset=UTF-8"/>
        <script type="text/javascript">
        

            // 打开子窗口并保存引用
            let childWindow = window.open('child.html', 'childWindow', 'width=600,height=400');

            // 动态发送消息给子窗口
            function sendMessageToChild(data) {
                // 确保子窗口存在且已加载
                if (childWindow && !childWindow.closed) {
                    childWindow.postMessage(data, '*');
                }
                childWindow.focus();  // 确保子窗口获得焦点
                self.blur();  // 父窗口失去焦点
            }

            // 监听子窗口的响应
            window.addEventListener('message', function(event) {
                if (event.source === childWindow) {
                    console.log('收到子窗口响应:', event.data);
                }
            });


        </script>
    </head>
 
    <body>
        <input type="button" value="获取选项的value值" onclick="sendMessageToChild('1111');"/>
    </body>
</html>

说明

最佳实践建议
安全性考虑‌：在使用postMessage时，建议指定具体的targetOrigin而不是使用’*’，以防止恶意网站截获消息
错误处理‌：在发送消息前检查子窗口是否仍然打开，避免在已关闭的窗口上操作
数据类型‌：postMessage可以传递各种JavaScript数据类型，包括对象、数组等复杂结构
性能优化‌：对于频繁的数据更新，可以考虑使用防抖或节流技术优化性能。
推荐使用postMessage方法‌，因为它提供了最好的安全性和灵活性，同时支持各种复杂数据类型的传递
